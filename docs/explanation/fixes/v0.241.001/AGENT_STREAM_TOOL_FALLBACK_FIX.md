# Agent Stream Tool Fallback Fix

Fixed in version: **0.239.201**

## Overview

Semantic Kernel agent streaming could fail for certain configured models even when the same model worked in direct model mode.

- Phi-4 agent streams failed with a `400` error indicating that `auto` tool choice required parser flags.
- Grok agent streams failed with a `431` header-size error during agent execution.

Both failures happened before the user received a useful streamed response, leaving the chat UI with a terminal `Agent streaming failed` message.

## Root Cause

The agent streaming path in `application/single_app/route_backend_chats.py` called `selected_agent.invoke_stream(...)` once and treated all failures as terminal.

- Semantic Kernel agents were initialized with `FunctionChoiceBehavior.Auto(maximum_auto_invoke_attempts=10)`.
- The streaming route had no retry path for models that rejected auto tool choice or could not tolerate the request size generated by tool-enabled agent calls.
- Because the failure occurred in the agent path, the recent streaming model-endpoint resolution fix did not address it.

## Technical Details

### Files Modified

- `application/single_app/route_backend_chats.py`
- `application/single_app/config.py`
- `functional_tests/test_agent_stream_tool_retry_fallback.py`

### Code Changes Summary

- Added a retry classifier for agent streaming failures that can recover with tool calling disabled.
- Added a temporary retry mode that clears the agent's `function_choice_behavior` and any attached execution-setting copies before the second attempt.
- Limited retries to a single retry-only path and only when the initial failure happens before any response content is streamed.
- Restored the original agent settings after the retry attempt completes.

## Testing Approach

Validation is covered with a functional regression test that verifies:

- Retry helper functions exist in the streaming route.
- Phi-style tool-choice failures are classified as retryable.
- Grok-style `431` header-size failures are classified as retryable.
- The stream route applies the retry once, disables tool calling for that retry, and restores the original agent state afterward.

## Impact Analysis

### Before

- Phi-4 agents failed immediately when Semantic Kernel requested `auto` tool choice.
- Grok agents failed immediately on the oversized tool-enabled agent request.
- Users saw a terminal stream failure with no built-in recovery attempt.

### After

- The streaming route retries once with tool calling disabled for the known retryable agent failures.
- Supported models keep the existing tool-enabled path on the first attempt.
- Agent retry behavior is isolated to the Semantic Kernel streaming path.

## Validation

- Added static regression coverage in `functional_tests/test_agent_stream_tool_retry_fallback.py`.
- The app version was updated in `application/single_app/config.py` to `0.239.201` to track the fix.
